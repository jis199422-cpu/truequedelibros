 Plan: Módulo Comunidad
 Contexto
 Los usuarios necesitan un espacio donde publicar ideas e imágenes, comentar y dar likes a las publicaciones de otros. El feed usa "formato burbuja" (posts con más interacción al principio) con lazy loading de 10 posts por
 página. Horarios en zona Argentina (UTC-3). MVP con posts, comentarios, respuestas (1 nivel), likes en posts y en comentarios.

 ---
 Backend — nuevo módulo comunidad/

 1. Migración V12

 Archivo: src/main/resources/db/migration/V12__comunidad_schema.sql

 CREATE TABLE posts (
     id            UUID         PRIMARY KEY,
     author_id     UUID         NOT NULL REFERENCES users(id),
     content       TEXT         NOT NULL,
     image_url     VARCHAR(500),
     like_count    INT          NOT NULL DEFAULT 0,
     comment_count INT          NOT NULL DEFAULT 0,
     created_at    TIMESTAMP    NOT NULL,
     updated_at    TIMESTAMP    NOT NULL
 );

 CREATE TABLE post_comments (
     id                UUID  PRIMARY KEY,
     post_id           UUID  NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
     author_id         UUID  NOT NULL REFERENCES users(id),
     content           TEXT  NOT NULL,
     parent_comment_id UUID  REFERENCES post_comments(id) ON DELETE CASCADE,
     like_count        INT   NOT NULL DEFAULT 0,
     created_at        TIMESTAMP NOT NULL,
     updated_at        TIMESTAMP NOT NULL
 );

 CREATE TABLE post_likes (
     id      UUID PRIMARY KEY,
     user_id UUID NOT NULL REFERENCES users(id),
     post_id UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
     CONSTRAINT uq_post_like UNIQUE (user_id, post_id)
 );

 CREATE TABLE comment_likes (
     id         UUID PRIMARY KEY,
     user_id    UUID NOT NULL REFERENCES users(id),
     comment_id UUID NOT NULL REFERENCES post_comments(id) ON DELETE CASCADE,
     CONSTRAINT uq_comment_like UNIQUE (user_id, comment_id)
 );

 CREATE INDEX idx_posts_bubble     ON posts (like_count, comment_count, created_at);
 CREATE INDEX idx_post_comments_pid ON post_comments (post_id);

 2. Entidades

 comunidad/domain/Post.java
 - UUID id (@GeneratedValue(strategy = UUID))
 - @ManyToOne(LAZY) User author
 - String content (@Column(columnDefinition="TEXT"))
 - String imageUrl (nullable)
 - int likeCount, int commentCount (contadores desnormalizados)
 - LocalDateTime createdAt, updatedAt
 - @PrePersist / @PreUpdate para timestamps

 comunidad/domain/PostComment.java
 - UUID id, @ManyToOne(LAZY) Post post, @ManyToOne(LAZY) User author
 - String content
 - @ManyToOne(LAZY) PostComment parentComment (nullable — 1 nivel de respuestas)
 - int likeCount
 - LocalDateTime createdAt, updatedAt

 comunidad/domain/PostLike.java
 - UUID id, @ManyToOne(LAZY) User user, @ManyToOne(LAZY) Post post
 - @Table(uniqueConstraints = @UniqueConstraint(columnNames = {"user_id","post_id"}))
 - Patrón idéntico a BookLike.java

 comunidad/domain/CommentLike.java
 - UUID id, @ManyToOne(LAZY) User user, @ManyToOne(LAZY) PostComment comment
 - @Table(uniqueConstraints = @UniqueConstraint(columnNames = {"user_id","comment_id"}))

 5. Servicios

 PostService (@Service @Transactional)
 - getFeed(User user, int page) → PostFeedPage
   - PageRequest.of(page, 10) con findAllBubbleSorted
   - Batch-check liked: postLikeRepository.existsByUserIdAndPostId por cada post (o query IN)
 - createPost(User user, CreatePostRequest req) → PostResponse
 - updatePost(UUID postId, User user, UpdatePostRequest req) → lanza AppException(FORBIDDEN) si no es autor ni admin
 - deletePost(UUID postId, User user) → idem control de autor
 - togglePostLike(UUID postId, User user) → guarda/elimina PostLike, actualiza post.likeCount ±1
 - getPostUploadUrl(String filename, String contentType) → llama S3StorageService igual que en BookService

 CommentService (@Service @Transactional)
 - getComments(UUID postId, User user) → List<CommentResponse> (árbol: raíces + replies anidadas)
 - createComment(UUID postId, User user, CreateCommentRequest req) → incrementa post.commentCount
 - updateComment(UUID postId, UUID commentId, User user, ...) → control de autor
 - deleteComment(UUID postId, UUID commentId, User user) → decrementa post.commentCount
 - toggleCommentLike(UUID commentId, User user) → guarda/elimina CommentLike, actualiza comment.likeCount ±1

 8. API client (comunidad.api.js)

 import client from '../../../shared/api/client'

 export const getPosts = (page = 0) => client.get(`/comunidad/posts?page=${page}`)
 export const createPost = (data) => client.post('/comunidad/posts', data)
 export const updatePost = (id, data) => client.put(`/comunidad/posts/${id}`, data)
 export const deletePost = (id) => client.delete(`/comunidad/posts/${id}`)
 export const togglePostLike = (id) => client.post(`/comunidad/posts/${id}/likes`)
 export const getPostUploadUrl = (filename, contentType) =>
   client.post('/comunidad/posts/upload-url', { filename, contentType })

 export const getComments = (postId) => client.get(`/comunidad/posts/${postId}/comments`)
 export const createComment = (postId, data) => client.post(`/comunidad/posts/${postId}/comments`, data)
 export const updateComment = (postId, cId, data) => client.put(`/comunidad/posts/${postId}/comments/${cId}`, data)
 export const deleteComment = (postId, cId) => client.delete(`/comunidad/posts/${postId}/comments/${cId}`)
 export const toggleCommentLike = (postId, cId) => client.post(`/comunidad/posts/${postId}/comments/${cId}/likes`)

 9. Lazy loading (patrón LikesPage)

 ComunidadPage usa IntersectionObserver sobre un sentinel <div ref={sentinelRef} /> al final del listado. Mantiene pageRef = useRef(0) y hasMore en estado. Appends nuevos posts al array existente.

     timeZone: 'America/Argentina/Buenos_Aires',
     day: '2-digit', month: '2-digit', year: 'numeric',
